## Начало работы с Python

### Введение в Python

Python (британское произношение: /ˈpaɪθən/; американское произношение: /ˈpaɪθɑːn/) — язык программирования, изобретенный голландцем Гвидо фон Россумом. В настоящее время это самый популярный и используемый язык программирования в мире. Python делает упор на читаемость кода и грамматическую простоту. По сравнению с C, C++ и Java, которые являются столь же влиятельными языками программирования, Python позволяет пользователям выражать свои намерения с помощью меньшего количества кода. Ниже приведены рейтинги Python, составленные несколькими авторитетными рейтингами языков программирования. Первая цифра предоставлена ​​TIOBE Index, а третья — IEEE Spectrum. Стоит отметить вторую картинку, которая демонстрирует популярность языков программирования на GitHub — крупнейшей в мире платформе для размещения кода. За последние четыре года чемпионский трон занял Python.

#### Хроники Питона

Вот некоторые важные моменты в развитии языка Python:

1. Декабрь 1989 г.: Гвидо ван Россум решил разработать новый язык сценариев и его интерпретатор, чтобы развеять скучное Рождество. Новый язык станет преемником языка ABC и будет в основном использоваться для замены оболочки Unix и языка C для управления системой. Поскольку сам Гвидо был большим поклонником сериала BBC «Летающий цирк Монти Пайтона», он выбрал слово Python в качестве названия нового языка.
2. Февраль 1991 г.: Гвидо ван Россум публикует первоначальный код интерпретатора Python в группе новостей alt.sources, обозначенный как версия 0.9.0.
3. Январь 1994 г.: Выпущен Python 1.0, с которого началась мечта.
4. Октябрь 2000 г.: выпущен Python 2.0. Весь процесс разработки Python стал более прозрачным, и экосистема начала постепенно формироваться.
5. Декабрь 2008 г.: выпущен Python 3.0, в котором реализовано множество новых функций современных языков программирования, но он не был полностью обратно совместим.
6. Апрель 2011 г.: впервые был выпущен pip, а язык Python получил собственный инструмент управления пакетами.
7 июля 2018 г.: Гвидо ван Россум объявляет о «постоянном уходе» с поста «Пожизненного великодушного диктатора» (человека, имеющего последнее слово при возникновении споров в сообществе проектов с открытым исходным кодом).
8. Январь 2020 г.: После 11 лет сосуществования Python 2 и Python 3 официально прекратили обновление и поддержку Python 2, надеясь, что пользователи как можно скорее перейдут на Python 3.
9. В настоящее время: Python широко используется в больших моделях (GPT-3, GPT-4, BERT и т. д.), компьютерном зрении (распознавание изображений, обнаружение объектов, генерация изображений и т. д.), интеллектуальных рекомендациях (YouTube, Netflix, ByteDance и т. д.), автономном вождении (Waymo, Apollo и т. д.), распознавании речи, науке о данных, количественной торговле, автоматизированном тестировании, автоматизированной эксплуатации и обслуживании, а экосистема языка Python также довольно процветает.

> **Примечание**: Номера версий большинства программ обычно делятся на три части, например A.B.C, где A представляет собой основной номер версии. A увеличивается только в случае полной переписывания и обновления программного обеспечения или при наличии несовместимых изменений. B представляет обновления функций. B увеличивается при появлении новых функций. C представляет собой незначительные изменения (например, исправление ошибки). C увеличивается до тех пор, пока есть модификация.

#### Преимущества и недостатки Python

У языка Python много преимуществ, и я просто перечислю вам некоторые из них.

1. **Простой и элегантный**, по сравнению со многими другими языками программирования, Python **легче для начала**.
2. Возможность делать больше с меньшим объемом кода, **повышение эффективности разработки**.
3. Открытый исходный код с **сильным сообществом и экосистемой**.
4. **Может делать многое** и обладает высокой способностью к адаптации.
5. **Язык-связь**, который может склеивать вещи, разработанные на других языках.
6. Интерпретируемые языки проще использовать на разных платформах и могут работать в нескольких операционных системах.

Основным недостатком Python является его низкая эффективность выполнения (распространенная проблема интерпретируемых языков). Если вы больше цените эффективность выполнения кода, то лучшим выбором для вас может стать C, C++ или Go.

### Установка среды Python

Если вы хотите хорошо выполнять свою работу, вам сначала нужно заточить свои инструменты. Если вы хотите начать свой путь программирования на Python, вам сначала необходимо установить среду Python на свой компьютер. Проще говоря, это означает установку интерпретатора Python, необходимого для запуска программ Python. Мы рекомендуем вам установить официальный интерпретатор Python 3, написанный на языке C и обычно называемый CPython. Возможно, на данный момент это ваш лучший выбор. Сначала нам нужно найти ссылку для скачивания на [странице загрузки] официального сайта (https://www.python.org/downloads/). После нажатия кнопки «Загрузить» для перехода на страницу загрузки вам необходимо выбрать соответствующую программу установки Python 3 в соответствии с вашей операционной системой, как показано на рисунке ниже.

<img src="res/day01/python_download_page_1.png" style="zoom:40%;">

После входа на страницу загрузки некоторые версии Python не предоставляют установщики для систем Windows и macOS, а предоставляют только сжатые файлы исходного кода. Для тех, кто знаком с системами Linux, мы можем собрать и установить их из исходного кода; Тем, кто использует системы Windows или macOS, мы по-прежнему настоятельно рекомендуем использовать установщик. Например, если вы хотите установить Python 3.10, выберите Python 3.10.10 или Python 3.10.11, чтобы найти установочный пакет для Windows или macOS, в то время как другие версии могут иметь только исходный код, как показано на рисунке ниже.

#### Окружение Windows

Ниже мы рассмотрим на примере Windows 11, как установить среду Python в операционной системе Windows. Дважды щелкните, чтобы запустить установщик, загруженный с официального сайта, и откроется мастер установки, как показано на рисунке ниже.

<img src="res/day01/install_python_1.png" style="zoom:50%;">

Во-первых, обязательно отметьте опцию «Добавить python.exe в PATH», которая поможет нам добавить интерпретатор Python в переменную среды PATH системы Windows (не беда, если вы ее не понимаете, просто проверьте); во-вторых, «Использовать права администратора при установке py.exe» — это получение прав администратора в процессе установки, рекомендуется его отметить. Затем выбираем «Настроить установку» и используем режим выборочной установки. Это выбор профессионалов, и вы (притворяетесь) этим профессионалом. Не рекомендуется использовать «Установить сейчас» (установка по умолчанию).

Далее мастер установки предложит вам отметить необходимые «Дополнительные функции». Здесь мы можем напрямую выбрать все. Стоит отметить, что второй пункт — это инструмент управления пакетами Python pip, который может помочь нам установить сторонние библиотеки и инструменты, поэтому обязательно проверьте его и нажмите «Далее», чтобы перейти к следующему шагу.

<img src="res/day01/install_python_2.png" style="zoom:50%;">

Далее следует выбор «Дополнительных параметров». Здесь мы рекомендуем вам отметить только два параметра: «Добавить Python в переменные среды» и «Предварительно скомпилировать стандартную библиотеку». Первый вариант поможет нам автоматически настроить переменные среды, а второй выполнит предварительную компиляцию стандартной библиотеки (сгенерирует файл `.pyc`), поэтому нет необходимости временно компилировать ее при использовании. Как я уже сказал, неважно, если вы не понимаете, просто поставьте галочку. Настоятельно рекомендуется изменить «Настроить место установки» ниже на пользовательский путь. Этот путь не должен содержать китайские иероглифы, пробелы и другие специальные символы. Если вы обратите на это внимание, то избавите себя от множества ненужных проблем в будущем. #### Среда macOS

#### Другие способы установки

Некоторые люди могут порекомендовать новичкам установить [Anaconda](https://www.anaconda.com/download/success) напрямую, поскольку Anaconda поможет нам установить интерпретатор Python и некоторые часто используемые сторонние библиотеки. Кроме того, он также предоставляет несколько удобных инструментов, которые особенно подойдут новичкам. Я лично не рекомендую этот метод, поскольку при установке Anaconda вы необъяснимым образом установите множество полезных и бесполезных сторонних библиотек (занимая больше места на жестком диске), а затем ваш терминал или командная строка будут испорчены Anaconda (автоматически активируя виртуальную среду при каждом запуске), что не соответствует **принципу наименьшего удивления** при проектировании программного обеспечения. Другие незначительные проблемы с Anaconda здесь обсуждаться не будут. Если вам необходимо использовать Anaconda, Рекомендуется установить Miniconda, которая находится на той же странице загрузки, что и Anaconda.

Новички часто слышат или говорят: «Я хочу писать программы на Python, разве я не могу просто установить PyCharm?» Вот краткое введение. PyCharm — это всего лишь инструмент, помогающий писать код на Python. Он не имеет возможности самостоятельно запускать код Python. Выполнение кода Python зависит от интерпретатора Python, который мы установили выше. Конечно, некоторые версии PyCharm предложат вам загрузить интерпретатор Python онлайн, если они не смогут обнаружить среду Python на вашем компьютере при создании проекта Python.
